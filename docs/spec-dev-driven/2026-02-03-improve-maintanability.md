# Améliorer la maintenabilité du code

## Contexte

Actuellement, tout le projet est "vibe-codé", mais on n'a pas vraiment fait attention à la maintenabilité du code. Pour améliorer cela, on doit suivre des conventions de style et d'architecture des fichiers.

## Spécifications techniques

Lis le fichier `docs/technical-notes/ts-react-style-guide.md` que je viens de créer. Il contient les conventions de style et d'architecture des fichiers que nous devons suivre pour assurer la maintenabilité du code.

## Tâches

- [ ] Lire et comprendre le guide de style et d'architecture des fichiers dans `docs/technical-notes/ts-react-style-guide.md`.
- [ ] Appliquer ces conventions de style et d'architecture dans tout le code existant
      On va appliquer ces conventions une par une, ce qui nous permettra de faire un commit pour chaque convention appliquée.
